# Tools Worth Knowing About

A short list of tools that are genuinely useful for research, studying, and organising ideas — not another "top 47 AI tools" roundup. These are specific enough to actually be helpful.

Most of the students I've seen fall behind are spending time collecting certificates and bookmarking tools. The ones moving faster are using a small set of tools actually well, not hoarding options.

---

## Writing & Research

**[Overleaf](https://www.overleaf.com/)**
Online LaTeX editor. The standard tool for writing research papers, IEEE submissions, and academic reports. No local install needed — collaborative, version-controlled, and renders LaTeX in real time. If you're submitting to a conference or journal, you'll need this.

Jake's Resume template (clean single-column LaTeX): https://www.overleaf.com/latex/templates/jakes-resume/syzfjbzwjncs — worth using for a resume that parses cleanly through ATS.

**[Elicit](https://elicit.com)**
For finding and screening research papers without drowning in Google Scholar results. You ask a question in natural language and it pulls relevant papers, summarises each one, and lets you compare findings across sources. Use this when you need to cite something credible fast, or when you're scoping a topic for a project and need to understand the existing literature.

**[SciSpace](https://typeset.io)**
Paste in a difficult research paper and ask it questions. It explains confusing sections in plain language and can walk you through the methodology. Use this when you've found the paper you need but the notation and terminology are getting in the way.

## Capturing and organising ideas

**[Glasp](https://glasp.co)**
Highlight useful passages from YouTube transcripts, blog posts, and articles — then export those highlights as notes, structured summaries, or raw text. Useful if you're reading a lot of content across different sources and want one place to pull insights from later without re-reading everything.

**[MyMind](https://mymind.com)**
Save screenshots, links, images, quotes, and references without sorting them into folders. The search is good enough that you can find things by describing what you saved rather than remembering where you put it. Better than browser bookmarks for anything you might actually want to find again.

**[Tana](https://tana.inc)**
A note and knowledge system for people who think in connected ideas rather than linear lists. Better than Notion for anything that involves building up a knowledge base over time where you want to see connections between things. Has a learning curve but pays back if you use it consistently.

## Thinking and planning

**[Goblin Tools](https://goblin.tools)**
Breaks big vague tasks into small concrete steps. Useful when you're stuck at "I need to build a RAG system" and can't figure out where to start. Run it once to get unstuck, then work from the breakdown it gives you.

**[Flowith](https://flowith.io)**
Maps ideas visually instead of in a straight line. Better than a chat interface for planning something complex — a project architecture, a content plan, a research direction. Each node in the canvas can branch independently rather than forcing everything into a single thread.

---

## A note on tools in general

None of these are magic. The value is in removing friction from something you're already doing — finding papers, capturing ideas, breaking down tasks. If you're not actually doing those things, the tools don't help.

The pattern that works: pick one tool that removes friction from a real bottleneck in your current workflow. Use it until it's automatic. Then pick the next one.

Adding six tools at once means you spend two weeks learning tools instead of doing the thing you were supposed to do faster.
